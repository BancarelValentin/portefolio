RPI2 3 Domotique-Developpement

Le domainde de la domotique étant je trouve trés interessant au vu des possibilités qu' il pourrait nous apportait, j'ai décidez de m'y lancer por executer quelques tache simples automatiser.

Pour commencer dans ce domaine aux possibilitées quasi infini, j'ai mis en place un petit projet, me faire reveiller tout les matins par mon ordinateur, pour ce faire, ce dernier devra me jouer un morceau de musique de mon choix, allumer un lumiere et me faire couler un petit cafe avant meme que je me reveille bien sure. Et pouvoir controller le tout via une interface web.

Ne s'achant absolument pas d'où j'allais partir ni meme comment j'allais pouvoir controler les ports GPIO du RPI, j'ai donc fait quelques recherche et je suis tomber sur le blog d'un certain "Idleman" (voir source) qui m'aura bien aider pour débuter ce projet. En effet il met a disposition tout le code qui lui a êrmis de realiser un projet similaire. Voulant avoir quelque chose de plus personaliser, j'ai don recuperer seulement quelques lignes de son code notament la fonction PHP exec() permettant d'executer une ligne de commande dans le terminal. Bref apres quelques heures de developpement, me voila avec une interface web permettant de controler les ports GPIO de mon RPI ainsi que "reveil en ligne" dont le fonctionnement est expliquer un peu plus bas. Le code source de l'interface web et de tout ce portefolio sera bientot disponible dans les sources un peu plus bas.

Le reveil:
Dans un premier temps, le reveil était developper rapidement et pas tres proprement, je stocker les differents reveil (nom, heure, minutes, jour de la semaine et le morceau a jouer) puis j'ouvrais une page web "reveil" qui se rechargait toutes les 30 secondes et comparer la date et l'heure a celles presentes dans la base de données, si il y avait une correspondance alors j'activait tous les ports GPIO de mon raspberry et inserer une balise iframe (recuperer sur www.soundcloud.com ou n'importe quels autre site de musique permettant d' embed leurs cntenus) pour jouer une musique choisi dans la base de données. Par la suite, je prevois de mettre a jour ce code pour utiliser des listener ou quelque chose de moins lors que des requetes http toutes les 30 secondes, si cela est possible.

head.png 2014-02-17 00:25:37 S2 2 Developpement applications de gestion d'archive

Lors de mon stage de deuxième année, mon objectif principale était le développement d'une application PHP permettant aux clients de la société Michel Visy Archivage de gérer les stocker dans les locaux de l'entreprise. Voila la situation qui m'étit imposer, l'application doit permettre la connexion de different utilisateurs, chaque utilisateurs a le droit de consulter les archives de un ou plusieurs organismes clients de MVA, tel un site de e-commerce, l'application proposera un systeme de panier ou l'utilisateurs pourra ajouter des archives afin de les commander. De plus, tout les clients de MVA ne souhaiterons pas effectuer des recherches sur leurs archives selon les memes critères (ex: là ou une banque souhaitera rechercher des archives en fonction du numero de contrat, un hopital lui les rechercheras par numero de dossier patient).

La recherche d'archives est découper en plusieurs parties:
  • Selection de l'organisme cible de la recherche,
  • Formulaires de recherche,
  • Affichage des resultats avec possibilitées d'ajout au panier,
L'administrateur lui en plus de la recherche, il pourra:
  • Ajouter/supprimer des utilisateurs/organismes,
  • Renseigner/modifier les criteres de recherche d'un organisme,
  • Effectuer le suivi des commmandes en cours (visualisations,traitement,numerisation de documents,...),
  • Importer des données aux formats .CSV pour un organisme données,
  • Modifier les droits d'un utilisateur sur tel ou tel client.
